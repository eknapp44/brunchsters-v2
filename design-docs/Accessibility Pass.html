Keyboard navigation

Everything interactive should be reachable + visible-when-focused.

Fixed

Focus rings on every interactive element

styles/a11y.css

What was wrong

Buttons, options, list cards, tabs had no visible :focus-visible state — keyboard users couldn't see what they'd just tabbed to.

What I did

Added a 3-pixel accent-colored outline on focus-visible for .bs-btn, .bs-option, .ms-tab, list cards, and inputs. Suppressed click-focus separately so mouse users don't see rings.

CancelChangeModals.jsx

Bottom sheet dismissable by keyboard

MobileShell.jsx

Before

The mobile quick-brunch bottom sheet dismissed by drag or scrim tap — no keyboard equivalent.

After

Fixed

Focus trap inside modals + bottom sheet

useFocusTrap.jsx, ModalShell, BottomSheet

Before

When a modal or sheet was open, Tab leaked to the page underneath. Real defect for screen-reader users.

After

Built a useFocusTrap(ref, isOpen) hook that snapshots the previously-focused element, focuses the first focusable inside the container on open, wraps Tab/Shift+Tab cycling, and restores focus to the trigger on close. Wired into ModalShell (all six modals: cancel, change, sign-out, delete account ×2, photo upload, share sheet) and BottomSheet (mobile quick brunch).

Fixed

Decorative SVGs no longer pollute screen readers

Icons.jsx, IconsPop.jsx, CountdownRing.jsx

Before

Every Pop icon was announced as "image" — e.g. "image, image, image, Sun May 3 11:00 AM." Noisy.

After

Added aria-hidden="true" + focusable="false" to every decorative icon. Pass an optional label prop to opt-in to role="img" + announcement when needed.

Fixed

Wizard progress bar announces step changes

PlateProgressBar.jsx

Before

The breakfast-metaphor bar was just a bunch of divs — screen readers saw nothing.

After

Wrapped in role="progressbar" with aria-valuemin/max/now and a aria-label like "Step 2 of 4: Where". Updates as the user advances.

Toggle switches use proper switch role

MeTab.jsx

Before

Custom button styled as a switch with no semantics — VoiceOver said "button" not "switch, on / off".

After

Fixed

Avatar accessible names

.bs-avatar across all screens

Before

"MR" announced as the literal text "MR" without context. Most avatars used title which is unreliable on mobile + for SR users.

After

Avatars in stacks where the name isn't shown beside them now use aria-label={name} + role="img" (dashboard hero stack, mobile hero strip). Avatars in rows that already show the name as text are marked aria-hidden="true" so they don't double-announce. TopBar profile avatar uses aria-label="Yusef O. — profile".

Motion & animation

Respect users who get motion sickness or vestibular triggers.

Fixed

Reduced motion is honored everywhere

styles/a11y.css

Before

Confetti burst on success, bobbing loader, slide-up sheets, drawer animation — all ran regardless of OS preference.

After

A global @media (prefers-reduced-motion: reduce) rule kills animations + transitions across the app. Motion-sensitive users get a static UI.

Selected state doesn't rely on color alone

.bs-option.selected

Issue

Selected voting options were communicated only via accent color + filled radio. Color-blind users could miss the state.

After

Selected options now have a bolder border + box-shadow outline as a non-color reinforcement.

Fixed

Windows High Contrast mode supported

styles/a11y.css

Why

Forced-colors mode strips our custom palette — without an opt-in, buttons go invisible.

Fix

@media (forced-colors: active) remaps our buttons + pills + progress to CanvasText / Highlight system colors so they survive.

Fixed

Visually-hidden helper class

.sr-only

Why

Some labels we want SRs to hear but not show ("Cast your vote", around an icon-only button). Standard pattern; we didn't have one.

Added

A .sr-only class that visually hides content but keeps it in the SR tree.
